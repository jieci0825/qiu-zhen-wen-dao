# == 和 === 有什么区别？

- ==（宽松相等）：会在比较两个操作数时执行 类型转换，尝试将两者转换为相同类型后再比较。
- ===（严格相等）：不会执行类型转换，仅在类型和值完全相同的情况下返回 true。
- 推荐使用 ===：因为它更严格、更符合预期，能避免潜在的错误。尤其是在需要精确判断值和类型时。
- 实际工作中，使用 if (a == null) 可判断 a 是否是 null 或者 undefined。

**下面列举一些常见的对比结果**：

|--------------------------|--------------------------|-------|-------|

| `true`                     | `true`                     | ✅    | ✅    |
| `false`                    | `false`                    | ✅    | ✅    |
| `'foo'`                    | `'foo'`                    | ✅    | ✅    |
| `0`                        | `0`                        | ✅    | ✅    |
| `+0`                       | `-0`                       | ✅    | ✅    |
| `+0`                       | `0`                        | ✅    | ✅    |
| `-0`                       | `0`                        | ✅    | ✅    |
| `0n`                       | `-0n`                     | ✅    | ✅    |
| `0`                       | `false`                    | ✅    | ❌    |
| `""`                       | `false`                    | ✅    | ❌    |
| `""`                       | `0`                        | ✅    | ❌    |
| `'0'`                      | `0`                        | ✅    | ❌    |
| `'17'`                     | `17`                       | ✅    | ❌    |
| `[1, 2]`                   | `'1,2'`                    | ✅    | ❌    |
| `new String('foo')`        | `'foo'`                    | ✅    | ❌    |
| `null`                     | `undefined`                | ✅    | ❌    |
| `null`                     | `false`                    | ❌    | ❌    |

| `{ foo: 'bar' }`           | `{ foo: 'bar' }`           | ✅    | ❌    |
| `new String('foo')`        | `new String('foo')`        | ✅    | ❌    |
| `0`                        | `null`                     | ❌    | ❌    |
| `0`                        | `NaN`                      | ❌    | ❌    |
| `'foo'`                    | `NaN`                      | ❌    | ❌    |
| `NaN`                      | `NaN`                      | ❌    | ❌    |
