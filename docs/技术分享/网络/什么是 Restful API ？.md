# 什么是 Restful API ？
**RESTful API 是一种软件架构风格**，用于设计网络应用程序的接口。主要特点：

**资源导向**
- 使用 URL 定位资源
- 每个资源都有唯一的 URL
- 资源可以有多种表现形式（如 JSON、XML）

**HTTP 方法对应操作**
- GET：获取资源
- POST：创建资源
- PUT：更新资源（完整更新）
- PATCH：更新资源（部分更新）
- DELETE：删除资源

**无状态**
- 服务器不保存客户端状态
- 每个请求包含所需的所有信息
- 有利于横向扩展

**统一接口**
- 使用标准的 HTTP 方法
- 使用标准的 HTTP 状态码
- 返回格式一致（通常是 JSON）
